# Описание проекта: Виджет банковских операций клиента
## О проекте
Проект содержит набор функций для обработки и отображения банковских операций клиента. 
Реализована возможность маскировки информации, 
фильтрации и сортировки операций

## Установка
### Требования
- Python 3.13

## Функции
### Маскировка данных
### get_mask_card_number
Частично маскирует номер банковской карты
#### Параметры
- card_number(str): номер карты
#### Возвращаемое значение
- str: замаскированный номер карты
#### Пример
`7000 79 ** **** 6361`

### get_mask_account
Маскирует номер банковского счета
#### Параметры
- account_number(str): номер счета
#### Возвращаемое значение
- str: замаскированный номер счета
#### Пример
`Счет **4305`

### mask_account_card
Маскирует одновременно номер счета и карты
#### Параметры
- account_number(str): номер счета
- card_number(str): номер карты
#### Возвращаемое значение
- str: (замаскированный номер счета или замаскированный номер карты)
#### Пример
`Счет **4305`

### get_date
Форматирует дату в нужный формат
#### Параметры
- date_str(str): строка с датой в формате ISO
#### Возвращаемое значение
- str: отформатированная дата в формате ДД.ММ.ГГГГ
#### Пример
get_date("2024-03-11T02:26:18.671407")
`"11.03.2024"`

### Работа с операциями
### filter_by_state
Фильтрует список операций по значению ключа
#### Параметры
- list_of_dict(list): список словарей
- state(str): значение ключа для фильтрации (по умолчанию - ‘EXECUTED’)
#### Возвращаемое значение
- list: отфильтрованный список операций
#### Пример
#### Выход функции со статусом по умолчанию 'EXECUTED'
[{'id': 41428829, 'state': 'EXECUTED', 'date': '2019-07-03T18:35:29.512364'}, {'id': 939719570, 'state': 'EXECUTED', 'date': '2018-06-30T02:08:58.425572'}, {'id': 594226727, 'state': 'CANCELED', 'date': '2018-09-12T21:27:25.241689'}, {'id': 615064591, 'state': 'CANCELED', 'date': '2018-10-14T08:21:33.419441'}]

### sort_by_date
Сортирует список операций по дате
#### Параметры
- operation_list(list): список словарей с операциями
- reverse_sort(bool): порядок сортировки - по убыванию
#### Возвращаемое значение
- list: отсортированный список операций
#### Пример
#### Выход функции (сортировка по убыванию)
[{'id': 41428829, 'state': 'EXECUTED', 'date': '2019-07-03T18:35:29.512364'}, {'id': 615064591, 'state': 'CANCELED', 'date': '2018-10-14T08:21:33.419441'}, {'id': 594226727, 'state': 'CANCELED', 'date': '2018-09-12T21:27:25.241689'}, {'id': 939719570, 'state': 'EXECUTED', 'date': '2018-06-30T02:08:58.425572'}]

### filter_by_currency
Принимает на вход список словарей, представляющих транзакции
#### Параметры
- usd_transactions(list): список транзакций
#### Возвращаемое значение
-list: итератор, который поочередно выдает транзакции, где валюта операции соответствует заданной
#### Пример
` {
        "id": 41428829,
        "operationAmount": {
            "amount": "1000",
            "currency": {
                "code": "USD"
            }
        },
        "description": "Перевод организации"
    }`

### transaction_descriptions
Принимает список словарей с транзакциями и возвращает описание каждой операции по очереди
#### Параметры
- usd_transactions(list): список транзакций
#### Возвращаемое значение
-list: итератор, который возвращает описание каждой операции по очереди
#### Пример
`см выше`

### card_number_generator
Выдает номера банковских карт в формате XXXX XXXX XXXX XXXX, где X — цифра номера карты
#### Параметры
- card_number(str): номер карты
#### Возвращаемое значение
-list: итератор, который выдает номера банковских карт в формате XXXX XXXX XXXX XXXX
#### Пример
`"5454 6200 1103 9772"`

### decorators
Передает аргументы в декоратор для настройки его поведения
#### Параметры
Позволяют настраивать его поведение извне при вызове, передавая значения как аргументы в скобках
(декоратор с аргументами -> декоратор -> обертка)
#### Важно
Для того, чтобы декораторы работали корректно, необходимо соблюдать порядок их вызова.
Первым вызывается декоратор, расположенный ближе всего к функции.
Каждый следующий декоратор — обертка для предыдущего.
Функция выполняется снизу вверх — начиная с последнего декоратора
#### Возвращаемое значение
- функция (выполнение)
#### Пример
`"divide ok, result: 10.0, time: 0.000012 sec
divide error: ZeroDivisionError: division by zero (args=(30, 0), kwargs={})"`

## Тесты
Проект включает набор тестов для проверки корректности работы функций
### Запуск тестов
Для запуска тестов выполните:
`pytest`
### Структура тестов
Тесты расположены в директории tests/ и в идеале включают проверки для:
* основных функций
* граничных случаев
* обработки ошибок
### Требования
Для запуска тестов необходимы:
`pytest`

`pytest-cov`
### Покрытие тестами
Для проверки покрытия используйте:
`pytest --cov`

Для генерации отчета в HTML-формате:
`pytest --cov=src --cov-report=html`

Для определения конкретной непротестированной строки:
`pytest --cov-report term-missing --cov=src`
